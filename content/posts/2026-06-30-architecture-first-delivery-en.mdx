# Architecture First

Under pressure, teams often skip architecture and jump into sprints. It looks like progress — until API coupling, permission gaps, deployment drift, and UX state chaos show up mid-flight.

## Four questions I ask first

1. **Where data originates and lands** — single source of truth
2. **Who can do what in which state** — permission and audit boundaries
3. **How the system degrades on failure** — not just happy-path demos
4. **How we verify done** — measurable completion criteria

## Practical outputs

Within about a week you can have: system context, core API contracts, milestones, and a risk register. The goal is one shared map for product, design, and engineering.
