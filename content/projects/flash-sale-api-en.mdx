# Flash Sale API — High-Concurrency Limited Stock Backend

## Screenshot
![Flash Sale API Web](/og/labs/flash-sale-api/web.png)

**Tags**: NestJS, PostgreSQL, Redis, RabbitMQ, Jest

This Lab focuses on limited-stock flash-sale flows under extreme traffic. The system prevents overselling while keeping inventory consistency and service stability.

## Core Design
- Row-level locking with `SELECT ... FOR UPDATE` to handle race conditions
- RabbitMQ queue buffering to protect the database from traffic spikes
- Redis caching to accelerate reads and reduce DB pressure
- Rate limiting to control malicious or abnormal traffic

## API Examples
- `POST /purchase`: place an order while guaranteeing no overselling
- `GET /products/:id`: query product inventory

## Stack
- Framework: NestJS modular architecture
- Database: PostgreSQL with TypeORM
- Queue: RabbitMQ with amqplib
- Cache: Redis
- Testing: Jest
