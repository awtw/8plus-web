# 架構先行

很多團隊在壓力下會跳過架構討論，直接進 sprint。短期看起來有進度，中期通常會遇到：API 耦合、權限漏洞、部署環境不一致、UX 狀態無法收斂。

## 我會先問的四件事

1. **資料從哪來、到哪去** — 單一真相來源在哪？
2. **誰能在什麼狀態做什麼** — 權限與審計邊界
3. **失敗時怎麼退化** — 不是 happy path demo
4. **怎麼驗證交付完成** — 可量測的 done 定義

## 實務產出

通常一週內可以產出：系統上下文圖、核心 API 契約、里程碑與風險清單。這些文件不是為了好看，而是讓 PO、設計、工程在同一張地圖上對話。

若你正在評估重構或新平台，歡迎 [預約](/booking) 30 分鐘對談。
