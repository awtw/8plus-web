## 0. 執行摘要

8plus 正從「前端開發服務商」演進為 **Architecture-led Engineering Partner（架構驅動的技術夥伴）**。CI 不應追逐 AI 同質化視覺（紫青粒子、泛稱 AI-powered），而應傳達：**可驗證的架構、可交付的系統、可被信任的體驗**。

| 層級 | 決策 |
|------|------|
| **品牌角色** | 技術決策夥伴，非外包工廠 |
| **視覺主線** | Cohere 企業克制（白底冷灰 + 單一藍色互動色） |
| **Hero 暗色島** | Cinema 光帶 + 銳利 Logo（沉浸式首屏，不污染全站 token） |
| **字體分工** | CohereText 權威標題 / Unica77 功能內文 / CohereMono 技術標籤 |

## 1. 品牌定位與 CI 角色對應

### 1.1 定位陳述

> **8plus 協助團隊把需求拆成可驗證的技術架構，導入 AI 到真實流程，並以優秀的使用者體驗把產品做成能交付、能維護、能被信任的系統。**

| 維度 | 內容 |
|------|------|
| **品類** | B2B 技術顧問 / 架構與交付陪跑 |
| **差異化** | 架構先行 → AI 工作流落地 → 體驗與交付一體 |
| **非定位** | 泛 AI 自動化代理商、純 UI 工作室、低價外包 |
| **公開服務** | Engineering & Product Consulting、Next.js Architecture、Booking 諮詢（[Services](https://8plus.app/services)） |

### 1.2 競品座標（台灣 / 亞洲）

| 類型 | 代表 | 8plus 區隔 |
|------|------|------------|
| AI Workflow 自動化 | [SpeedMVPs](https://speedmvps.com/ai-workflow-in/taiwan)、[YCP](https://ycp.com/lp/taiwan-ai-automation-consulting) | 重流程自動化 KPI；8plus 重 **架構決策 + 人機協作設計** |
| 企業 AI 平台美學 | Cohere、Linear 類 | 借鑑克制企業感，但敘事聚焦 **顧問陪跑與可驗證交付** |
| 傳統前端外包 | 眾多工作室 | 8plus 賣 **決策品質與系統壽命**，非工時 |

### 1.3 CI 元素 → 品牌角色對應表

| CI 元素 | 承載的品牌角色 | 使用者應感受到 |
|---------|----------------|----------------|
| **Logo（雙圓 + 斜切）** | 識別錨點、信任印章 | 穩定、幾何精確、可記憶 |
| **Display 字體（CohereText）** | 權威、宣言、決策高度 | 「這裡談的是架構級問題」 |
| **Body 字體（Unica77）** | 清晰、可執行、專業 | 讀得懂、做得出 |
| **Mono 標籤（CohereMono）** | 工程語境、流程節點 | 有方法論、有節奏 |
| **主色 Accent Blue** | 唯一行動召喚 | 克制、可信、非炫技 |
| **22px 圓角卡片** | 企業級容器 | 精緻、有秩序 |
| **Cinema Hero 暗色光帶** | Signature 動效時刻 | 現代、沉浸、技術深度（僅首屏） |

## 3. 目標受眾（TA）畫像

### TA-1：技術決策者 — 「架構焦慮的 CTO / VP Eng」

| 項目 | 描述 |
|------|------|
| **角色** | 10–80 人新創或 SMB 技術負責人 |
| **痛點** | 技術債、選型錯誤、AI POC 無法進生產 |
| **搜尋語** | 架構 review、Next.js 顧問、系統重構 |
| **決策因子** | 可驗證方法論、過往案例、溝通清晰度 |
| **CI 觸點** | 案例研究、Process 頁、部落格深度文、Booking |
| **文案語氣** | 直接、有邊界、少形容詞 |

### TA-2：產品負責人 — 「要快但要對的 Founder / PM」

| 項目 | 描述 |
|------|------|
| **角色** | 產品 0→1 或改版負責人 |
| **痛點** | 需求模糊、工程排程不透明、UX 與後端脫節 |
| **搜尋語** | MVP 架構、產品技術顧問、交付流程 |
| **決策因子** | Discovery 門檻低（30min）、價格區間透明 |
| **CI 觸點** | Hero lead、Pricing 區間、Contact / LINE |
| **文案語氣** | 結果導向、時間線清晰 |

### TA-3：工程管理者 — 「要建立規範的 Eng Manager」

| 項目 | 描述 |
|------|------|
| **角色** | 帶 3–15 人工程團隊 |
| **痛點** | Review 品質、AI 工具導入亂象、交付節奏 |
| **搜尋語** | code review 顧問、工程效率、AI 導入 |
| **決策因子** | Retainer / Workshop 模式、可複製流程 |
| **CI 觸點** | Services、文章 checklist、Process Steps |
| **文案語氣** | 務實、工具鏈具體 |

## 4. CI 需求規範書

### 4.1 設計原則（Must）

1. **克制即專業** — 全站最多一個 signature 動效區（首頁 Cinema Hero）
2. **襯線宣示、無襯線執行** — Display vs Body 嚴格分工
3. **單一互動色** — `#1863dc` 僅用於 CTA / hover / focus
4. **22px 簽名圓角** — 主卡片容器統一 `--radius-md`
5. **冷色調** — 禁止暖色主導（Apple 模式僅 Share 實驗島）
6. **RWD 優先** — 行動版先可讀，再談動效

### 4.2 禁用清單（Must Not）

- 全站霓虹 / 掃描線 HUD（已淘汰）
- 氣泡粒子 logo 主視覺（使用者明確反感）
- 到處 gradient 按鈕
- Inter 作為主標題字體
- `rounded-2xl`（16px）取代 22px 主卡片

### 4.3 無障礙與效能

| 項目 | 規範 |
|------|------|
| 對比 | 內文 `--fg` on `--bg` ≥ WCAG AA |
| 動效 | `prefers-reduced-motion` 關閉 canvas 動畫 |
| 觸控 | CTA 最小高度 44px |
| 效能 | Hero canvas 僅首屏；內頁靜態 |

### 4.4 雙語 CI

| 元素 | zh-TW | en |
|------|-------|-----|
| 品牌名 | 8plus（不翻譯） | 8plus |
| 標語 | 以架構驅動 / AI 與體驗落地 | Architecture-led / AI and product experience |
| Eyebrow | 開放預約諮詢 | Available for consulting |
| 語氣 | 專業、直接 | Confident, precise |

## 5. 應用範疇清單

### 5.1 Logo 系統

#### 幾何語意

SVG 標誌（`components/logo.tsx`）由 **兩圓一斜切** 構成，象徵：雙系統接合、架構張力、精確幾何。

#### 變體

| 變體 | 檔案 / 實作 | 使用情境 |
|------|-------------|----------|
| **Default** | SVG `var(--logo-mark)` | 淺色 Nav、淺底 |
| **Light / Inverse** | `logo-light-512.png`、SVG light | 暗色 Hero、Cinema 舞台 |
| **Mono** | `logo-mono-512.png`、SVG mono | 折色 mask、印刷、單色 |
| **Favicon** | 白底黑標 | 瀏覽器 tab |

#### 安全空間與尺寸

| 場景 | 最小寬度 | 四周淨空 |
|------|----------|----------|
| Header | 28px | ≥ 0.5× logo 高度 |
| Hero Cinema | 120–280px | ≥ 1× logo 高度 |
| Footer | 24px | ≥ 0.25× |

#### Logo 動效規範（Cinema）

- ✅ 光帶流動、折色 drift、shine sweep、入場 blur→清晰
- ❌ 粒子分解、氣泡、過度 glitch

### 5.2 Typography 系統

#### 字體家族

| Token | 字體 | 角色 |
|-------|------|------|
| `--font-display` | CohereText, Georgia | H1–H3、品牌標語 |
| `--font-body` | Unica77 Cohere Web, Inter | H4–H6、內文、UI |
| `--font-mono` | CohereMono, JetBrains Mono | Eyebrow、流程編號、標籤 |

#### 字重規則

- Display / Body 內文：**400**
- 按鈕 / 強調標籤：**500–600**
- 禁止內文 **700+**

### 5.3 Color 系統

#### 主色盤（Cohere 主線 — Light）

| Token | Hex | 角色 |
|-------|-----|------|
| `--bg` | `#ffffff` | 頁面底 |
| `--surface` | `#f8fbff` | 卡片底 |
| `--fg` | `#0b1020` | 主標題 |
| `--fg-2` | `#2e3b52` | 內文 |
| `--muted` | `#60708b` | 輔助說明 |
| `--border` | `#d7e0ef` | 容器邊框 |
| `--accent` | `#1863dc` | 唯一互動色 |
| `--accent-hover` | `#1557bf` | Hover |
| `--accent-active` | `#123f8a` | Active / Pressed |

#### Cinema Hero 暗色擴展（僅 `html[data-hero-theme="lumina"]` 或 `.cinema-hero`）

| 用途 | 色值 |
|------|------|
| 舞台底 | `#020408` → `#0b1424` 徑向漸層 |
| 光帶高光 | `#38bdf8` / `#60a5fa` |
| 標題漸層字 | `#f0f9ff` → `#60a5fa` |
| Logo glow | `rgba(56, 189, 248, 0.5)` |

#### 語意色

| Token | Hex | 用途 |
|-------|-----|------|
| `--success` | `#16a34a` | 成功狀態 |
| `--warn` | `#eab308` | 警告 |
| `--danger` | `#ef4444` | 錯誤 |

## 6. 視覺原子系統

### 6.1 網格系統（Grid）

#### 基礎單位

**8px** 基數；常用刻度：`8 · 12 · 16 · 20 · 22 · 24 · 32 · 40 · 56 · 60`

#### 版面容器

```css
.section-shell {
  max-width: var(--container-max); /* 1440px */
  padding-inline: 16px → 24px (sm) → 32px (lg);
}
```

#### 12 欄邏輯格（設計稿）

| 斷點 | 欄數 | Gutter | 用途 |
|------|------|--------|------|
| <640px | 4 | 16px | 單欄堆疊 |
| 640–1023px | 8 | 20px | 雙欄卡片 |
| ≥1024px | 12 | 24px | Hero 5+7、Bento 4 欄 |

#### Hero 編排（現行 Cinema）

| 斷點 | 配置 |
|------|------|
| Desktop | 文案 ~55% │ Logo 舞台 ~45% |
| Mobile | Logo 上 → 文案下；CTA 全寬 |

#### 區段垂直節奏

- 區段間：`56–80px`（`aurora-section` / `cinema-hero` 下緣）
- 卡片內距：`24–32px`

### 6.2 色彩規範 — 語意映射

| 語意 | Light | Dark (cohere) | 使用 |
|------|-------|---------------|------|
| `background` | `--bg` | `--bg` | 頁面 |
| `foreground` | `--fg` | `--fg` | 標題 |
| `muted` | `--muted` | `--muted` | 說明 |
| `primary` | `--accent` | `--accent` | CTA |
| `border` | `--border` | `--border` | 卡片 |

### 6.3 字體規範 — H1～H6 對照表

| 層級 | 語意角色 | 字體 | 尺寸 Token | 建議尺寸 | 行高 | 字距 | 字重 | 範例用途 |
|------|----------|------|------------|----------|------|------|------|----------|
| **H1** | 主 Hero 標語 | Display | `--text-4xl` | `clamp(2.75rem, 7vw, 4.5rem)` | 1.00–1.06 | -0.04em | 400–600 | 首頁「8plus」品牌行 |
| **H2** | 區段標題 | Display | `--text-3xl` | `48–60px` / `clamp(1.75rem, 4vw, 3rem)` | 1.08–1.12 | -0.03em | 400 | 「核心服務」「合作流程」 |
| **H3** | 子區塊 / 卡片主標 | Body | `--text-2xl` | `32px` / `1.5–1.75rem` | 1.2 | -0.02em | 500 | Pricing tier 名稱 |
| **H4** | 功能標題 | Body | `--text-xl` | `24px` / `1.25rem` | 1.3 | normal | 500 | Pillar 標題 |
| **H5** | 列表項 / 小節 | Body | `--text-lg` | `18px` / `1.125rem` | 1.4 | normal | 400 | 文章小標 |
| **H6** | 標籤 / Meta | Mono 或 Body | `--text-sm` | `14px` | 1.4 | +0.08–0.14em | 500 | Eyebrow、流程編號 `01` |

#### CSS 實作建議

```css
/* 建議新增語意 class，與現有 .display-title 對齊 */
.text-h1 { font-family: var(--font-display); font-size: clamp(2.75rem, 7vw, 4.5rem); line-height: 1.04; letter-spacing: -0.04em; }
.text-h2 { font-family: var(--font-display); font-size: clamp(1.75rem, 4vw, 3rem); line-height: 1.08; letter-spacing: -0.03em; }
.text-h3 { font-family: var(--font-body); font-size: var(--text-2xl); line-height: 1.2; font-weight: 500; }
.text-h4 { font-family: var(--font-body); font-size: var(--text-xl); line-height: 1.3; font-weight: 500; }
.text-h5 { font-family: var(--font-body); font-size: var(--text-lg); line-height: 1.4; }
.text-h6 { font-family: var(--font-mono); font-size: var(--text-sm); letter-spacing: 0.12em; text-transform: uppercase; }
```

#### 現有 utility 對照

| 現有 class | 對應層級 |
|------------|----------|
| `.display-title` | H1–H2 |
| `.aurora-section-title` | H2–H3 |
| `.body-lead` | Lead（非 H 層級，介於 H5 與 body） |
| `.eyebrow` / `.capsule` | H6 |
| `.cinema-hero-brand` | H1 變體（漸層） |

### 6.4 圓角原子

| Token | 值 | 用途 |
|-------|-----|------|
| `--radius-sm` | 8px | Dialog、小標籤 |
| `--radius-md` | **22px** | 主卡片（簽名） |
| `--radius-pill` | 9999px | 按鈕、Badge |

### 6.5 陰影與深度

| 層級 | 處理 |
|------|------|
| L0 平面 | 無陰影 |
| L1 卡片 | `var(--elev-ring)` 淡邊框 |
| L2 Hover | `var(--shadow-hover)` 藍色淡光 |
| Hero | 無卡片陰影；靠光帶與 vignette |

### 6.6 動效原子

| Token / 模式 | 值 | 用途 |
|--------------|-----|------|
| `--motion-fast` | ~150ms | Hover |
| `--motion-base` | ~250ms | 卡片 |
| Framer fade | 0.5–0.65s | 區塊入場 |
| Cinema canvas | 60fps RAF | 僅首屏 |
